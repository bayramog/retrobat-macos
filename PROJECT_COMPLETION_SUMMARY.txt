KEY FINDINGS
================================================================================

Current RetroBat Architecture:
├── RetroBuild.exe (.NET Framework, Windows-only)
├── InstallerHost.exe (Windows installer)
├── emulatorLauncher (.NET Framework, XInput/DirectInput)
├── EmulationStation (Windows custom build)
└── System Tools (7za.exe, wget.exe, curl.exe)

Target macOS Architecture:
├── RetroBuild (.NET 6+/8, cross-platform)
├── macOS Installer (.dmg/.pkg)
├── emulatorLauncher (.NET 6+/8, SDL3)
├── EmulationStation-DE (already cross-platform)
└── Native macOS Tools (Homebrew, native curl)

Main Challenges Identified:
1. emulatorLauncher complexity (C# Windows APIs → cross-platform)
2. 1000+ configuration files (Windows paths → Unix paths)
3. Emulator compatibility (some Windows-only)
4. Code signing and notarization (Apple requirements)

Solutions Designed:
1. Migrate to .NET 6+/8 with platform detection
2. Automated scripts for bulk config updates
3. Compatibility matrix + alternatives for unavailable emulators
4. Apple Developer ID + signing pipeline
